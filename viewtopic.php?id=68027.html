Hi,

I am currently using a HP Microserver as my home server but feel like it is being overkill for what I use it for.
I was previously using a Seagate Dockstar that was almost fine but just a bit underpowered.

This is what I do on my server that is running Debian:

  • Storage (4TB+2TB for redundancy with rsync)

  • Router (2xGb Eth), Firewall (iptables), DNS cache and DHCP (dnsmasq), Host filtering (Nintendo servers, ads)

  • SSH server (control and remote file access)

  • Samba server (max 2 users concurently)

  • Apache web server & mysql & PHP: ampache, wordpress, piwigo, static pages/objects. HTTPS with Letsencrypt. The web server is very low traffic: max amount of concurrent user must be 3 or 4.

  • Munin

  • Transmission

  • MPD (USB sound card)

  • Dropbox backups

  • Dynamic DNS (inadyn)

  • Beets (Python progam for music management)

  • fail2ban

  • external wifi AP (used to be hostapd with internal PCIe)

I saw that they now have routers with Cortex A9 CPU, USB3 and 256MB RAM so was considering a Netgear R7000 for example.

Could it replace my server with a USB3 HDD ? I would then resell my microserver and have a lower power, lower noise device with better wifi (currently using a 2.4GHz basic 150Mbps AP).

No guarantees all goes flawlessly.
Someone in parallel topic blamed his 200$ dual core arm router cant output more than 30 mbytes/sec thru samba while PC can do 100 mb/s.
You say you have gbit connection. If you really connected to inet with that speed or even 300 mbps then arm can be too slow to handle natting. If you also use qos or advanced filtering the margin is even lower.
So better keep your pc if possible.

If you don't need more than two ethernet ports and you are okay with basic wireless, I might suggest the Zotac ZBox CI323 as an option.

Add some minimal amount of RAM and then boot off an old laptop hard drive or even a USB flash drive.

There are some other similar low priced micro PCs from other vendors should this hardware not meet your needs.

I am not sure what SoC is in the Netgear R7000, but I have a similar spec/grade model from another brand and I suspect you might run into some amount of slowness with these CPUs. They are indeed much faster and more capable than some of the SoCs of olden days, but they are not quite up to the point where you can fully replace your home PC. Also, some of your packages like python might not be well supported under OpenWRT/LEDE.

I would suggest looking at some low-end Intel NUC-like systems, like that Zotac I mentioned. I have several laying around and find them to be pretty useful.
