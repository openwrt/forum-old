Post #3
hnyman
25 Feb 2013, 08:07

I had similar problem with my WNDR3800 with r35754, although in my case the already existing config had no effect.

I had built the firmware without "make clean" first, so I suspected a dirty build environment as a reason.
After make clean, the new build of then already updated r35759 brought the radios back on.

There was a version bump of the radio driver at r35753 and then later several fixes upto r35762. Eithr a dirty build was the reason for me, or alternatively, the fixes really fixed things.
