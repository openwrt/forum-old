For those unfamiliar with the device and it's prepackaged firmwares:

The Tor firmware sets up 2 SSIDs, 1 of which is a regular Wi-Fi network, and the other is a Wi-Fi network that routes you out the WAN to Tor.  The regular Wi-Fi SSID will allow non-Tor WAN traffic and also LAN traffic.  The Tor SSID will not allow LAN traffic.  The LAN port does not route to Tor.

The Tor on LAN firmware does the above, but also routes the LAN port to TOR.  Meaning, your only administrative connection is the regular Wi-Fi SSID.

Regarding your issue, this sounds like a similar issue: some kind of admin change causes the SSIDs to crap out.  Most of the time I was in the web interface making changes, I was on the non-Tor based Wi-Fi.  I was able to recover the unit by connecting to it via the LAN port.  However, with the Tor over LAN firmware, that's not an option; the only management interface is the regular Wi-Fi SSID.  But, since both SSIDs are not broadcasting, I can't get in to the management interface.

If I reset the unit back to factory defaults, I will likely start the process over again.

Obviously, I would like to use the tor over LAN firmware and change the SSID of the Tor Wi-Fi network.

I was able to make it work by SSH'ing into the router.  It took me a minute to figure out how to get the thing into failsafe, and I experienced the absurdity of vi, but here's how to recover it and change some stuff:

If you get into the failed state that I described in the OP, this will help you recover the unit and change the WLAN SSIDs.
1. Boot into failsafe. Link (scroll down to Failsafe mode).

2. Connect with telnet.

2.5 I think there's some commands you're supposed to automatically run in failsafe, but that's not clear.  I obviously didn't run any commands, though I did see some an error when I tried to do something.

3. Give it the command "reboot -f".

NOTE: At this point, the router SSID "OpenWRT" came back and I could connect to the Luci interface.  The root password was gone (maybe I didn't set it?), but the "tor" SSID was not broadcasting.  The feedback in Luci also said something was wrong with the tor SSID or it was disconnected.

4. Connect to the router with SSH.  The password will be the "root" password you defined in Luci.  If you haven't made a root password, you can't SSH; therefore, make a root password and then connect SSH.

5. Install a better text editor.  In this case, I used nano:

opkg update
opkg install nano

NOTE: You have to do the update to get nano.

6. Edit the file "wireless" in the directory /etc/config/

$ /etc/config# nano wireless

or

$ nano /etc/config/wireless

7. Change your wifi settings.  Double quotes works to encapsulate spaces and special characters.  I suspect that single quotes cause the crash (the tor network settings had single quotes, while the OpenWRT did not).

8. Save and exit

9. Reboot the router.  I used the command "reboot".

You should see the new settings take effect.  I connected and did a tor check and it succeeded.

This is a pain in the ass, especially since getting into failsafe was tricky (for me).  I'm thinking aobut getting a Bluetooth TTL adapter to see if I can rig up a cable to get to the serial port if/when I need it.  I think the USB port on this device should be able to supply enough power, and I can configure the pins to TX, RX, and GND.  The adapters seem to be $5 - $10 if you get one stateside, or dirt cheap if you import them.
