Not even close to NJ, but I will try to help anyway

I don't know if you are familiar with OpenWrt compilation, but I fear this is a prerequisite if you want to get access to the GPIOs.

Usually, LED registration is located into the machine-specific file "target/linux/ar71xx/files/arch/mips/ath79/mach-xxx.c".

For the TP-Link TL-WR703N for example, the blue system LED is on GPIO27 and looks like this in file "target/linux/ar71xx/files/arch/mips/ath79/mach-tl-wr703n.c":

#define TL_WR703N_GPIO_LED_SYSTEM    27
...
static struct gpio_led tl_wr703n_leds_gpio[] __initdata = {
    {
        .name        = "tp-link:blue:system",
        .gpio        = TL_WR703N_GPIO_LED_SYSTEM,
        .active_low    = 1,
    },
};

Given that your board is named "GS-OOLITE", the file should be named accordingly something like "target/linux/ar71xx/files/arch/mips/ath79/mach-gs-oolite.c"

To make it simple, just remove everything inside the inner braces related to GPIO23 from the structure, do a "make clean" and "make world", then flash the resulting firmware.
