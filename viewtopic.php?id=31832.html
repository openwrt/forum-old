mios

I'm the CTO at MIOS.  We understand the GPL well, and we are fully compliant, and strong advocates of FOSS.  The GPL license requires you to release code when you link to GPL software (ie statically link to a .a file or dynamically to an .so).  Our main engine, Luup, only links to libLua.  Lua is published under the MIT license -- not the GPL license -- and it allows linking.  We do not link to any GPL code.  OpenWRT is a Linux distro, like Fedora, Ubuntu, etc., and is released under the same license.  Running on a Linux distro, which doesn't involve linking, in no way requires you to be GPL.  Think about it...  Most bank ATM's run on Linux distro's.  Many car navigation systems.  And countless other products.  Just because Audi runs on Linux (like we do) doesn't mean Audi then has to give away the source code to their in-car entertainment/navigation system.

This is a serious point of confusion.

The GPL =can= reach out and contaminate other software, but it doesn't automatically contaminate everything running on whatever machine the GPL software happens to be running on.

Most commercial developers are aware of the viral nature of the GPL, and have been for 10 or 20 years (depending on how old they are ...) and usually take all the required steps to insure there is no contamination of their product because of the GPL's viral nature.  As an aside, many free software developers (I'm one -- this system is almost certainly running software I wrote) are aware of the way that the GPL stifles software development, because of fear of viral infection, and don't release software under the GPL or the LGPL.  In my case, I had software stolen by developers and redistributed under the LGPL, with all of my copyright and license statements removed.  I could care less, but as has been noted by others, it happens.  There have been GPL enforcement lawsuit over code I wrote and I wasn't named as a party, even though my properly-attributed code was part of what was misappropriated.

One of the reasons for the LGPL is because the "linking" problem with the GPL meant it was very unlikely any commercial vendors would use anything with a GPL license.  Stallman can scream bloody hell that people aren't happy about his virus, but some of us have bills to pay and don't get subsidized by others.

Another point is that even IF a library is under the GPL, if the library isn't required, the viral infection doesn't happen -- in other words, if there is a GPL'd library, and a non-GPL'd library, and the interface isn't unique to the GPL'd version, the linkage doesn't create a derived work.  Something from a GPL'd "work" has to be included for their to be any sort of derived work involved.

So people need to be careful before tossing around accusations that the GPL has been violated because some people who don't support the GPL are in a position to sue product distributors who've benefited from our work and sued for damages when others have violated =their= rights.

THAT said, I also support free and open software, I just don't believe the GPL is the correct vehicle because it does create this vigilante mentality.  I've contributed free software to the free software using community for something like 27 or 28 years.  I've contributed to products that are licensed under the GPL and LGPL, and I make very =careful= use of GPL'd and LGPL'd products.
