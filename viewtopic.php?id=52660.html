dabyd64
11 Sep 2014, 03:34

I see the problem..You used jffs2!
jffs2 is fully writable, so you can cause serious problems. You can't reset filesystem to defaults with this format. Only configs...
Deleting any critical system file will be fatal and permanent, in some you will have to use a serial cable to restore it.

Use the squashfs build!
Squash is read only, openwrt stores the important system files on it, and the changes are stored on other filesystem, it creates a jffs2 filesystem on the free flash space.
A factory reset erases that filesystem, and you are ready to start again.


Does sysupgrade work? If so, you are lucky.
Download and transfer the Squasfs build to /tmp
Try:

sysupgrade -v /tmp/openwrt-ar71xx-generic-tl-wr710n-v1-squashfs-sysupgrade.bin

If that works, after the reboot it will work again.

dabyd64
12 Sep 2014, 02:25

wait, maybe we can transfer "mtd" some way back to the router... checking out...

edit...I was able to extract the jffs2 image and extract the mtd file.
Download here: https://dl.dropboxusercontent.com/u/23958662/mtd.zip


Download the "factory" openwrt firmware: (Name should be openwrt-ar71xx-generic-tl-wa701n-v1-squashfs-factory.bin)

Transfer "mtd" to the router, into /tmp, then:

cd /tmp
cat mtd>/sbin/mtd
chmod 777 /sbin/mtd

After this, type mtd and see if it executed.
If not, try with this:

/sbin/mtd

In the case that any of them worked, use this command:

mtd -r write /tmp/openwrt-ar71xx-generic-tl-wa701n-v1-squashfs-factory.bin firmware

or

/sbin/mtd -r write /tmp/openwrt-ar71xx-generic-tl-wa701n-v1-squashfs-factory.bin firmware
