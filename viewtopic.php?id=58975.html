this make target fails:

tools/pkg-config/compile

internal glib fails to compile (version 2.32.4 according to it's configure.ac)

gutils.c:156:28: error: 'glib_dll' undeclared (first use in this function)
   if (!GetModuleFileNameW (glib_dll, wc_fn, MAX_PATH))

afaict glib_dll is declared in glib-init.h but it seems there are visibility issues (G_GNUC_INTERNAL docs) so checking compile flags or comparing might help
(try to compile self downloaded pkg-config 0.28 and see if the same error occurs and then compare the build cflags you see between OpenWrt and your try)

config.log and/or console might have a line

for GNUC visibility attribute

that might be different between working/nonworking

i have:

checking for ISO C99 varargs macros in C... yes
checking for ISO C99 varargs macros in C++... yes
checking for GNUC varargs macros... yes
checking for GNUC visibility attribute... yes
checking whether using Sun Studio C compiler... no

in the configuration step of pkg-config/glib


edit: current glib source is at https://git.gnome.org/browse/glib/tree/glib/gutils.c
the GNUC visibility macro stuff (in that .h file) was removed in later glib versions

Congratulations - it seems there is a bug (at least with that configure switch) in pkg-config.

Your alternatives are:

You could try to build pkg-config from latest git (with the same ./configure --with-internal-glib):

git://anongit.freedesktop.org/pkg-config

internal glib of that snapshot is  2.38.2 - i think its fixed there (its has the visibility changes afaict in alternative 3)

Install at least some glib-2.0 version on cygwin and remove 

HOST_CONFIGURE_ARGS += --with-internal-glib

from tools/pkg-config/Makefile and try if OpenWrt can build that way

Check the configure line regarding visibility since newer glib seems to handle this differently (thats why the git pkg-config might work with internal glib 2.38.2) ->
remove the G_GNUC_INTERNAL from glib-init.h like it is done in
- https://git.gnome.org/browse/glib/commi â€¦ d3864ec744

(there are other patches that are possibly part of this series like "win32: visibility fixups", "visibility: Use a separate CFLAGS variable", "Fix visibility for glib/ and gio/ submodules"
