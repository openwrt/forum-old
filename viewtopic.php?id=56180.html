Same with my TL-WR740n v4 running AA (12.09-rc1, r34185). I made a workaround by running a script with cron every minute that restarts WLAN, if it is down.

iwinfo wlan0 info | sed -n 5,5p | grep -q '^          Signal: unknown' && (logger wifi down, restarting;wifi up)

micha123 wrote:

Ideas why Lietas script sample didn't work (besides the first iwinfo instatt iw)?!

iwinfo and iw are completely different utilities.

root@Lieta_TP_link:~# which iwinfo
/usr/bin/iwinfo
root@Lieta_TP_link:~# which iw
/usr/sbin/iw
root@Lieta_TP_link:~# iwinfo wlan0 info
wlan0     ESSID: "wlan name"
          Access Point: 64:70:02:7C:36:48
          Mode: Master  Channel: 1 (2.412 GHz)
          Tx-Power: 17 dBm  Link Quality: 0/70
          Signal: unknown  Noise: -90 dBm
          Bit Rate: unknown
          Encryption: WPA2 PSK (CCMP)
          Type: nl80211  HW Mode(s): 802.11bgn
          Hardware: unknown [Generic MAC80211]
          TX power offset: unknown
          Frequency offset: unknown
          Supports VAPs: yes

The script searches for "Signal: unknown" and restarts wifi, if found.
I just found that "iwinfo wlan0 info" prints "Signal: unknown" if wifi is up, but nobody has connected to it. So probably not the best way to check wifi signal availability.
