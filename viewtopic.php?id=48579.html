The corrrect failsafe trigger moment is quite dependent on the router model, so you need exactly correct advice. Advice for other routers does not help much.

But in general, the boot process goes:
1) bootloader (u-boot etc.) = "hardware BIOS". Kicks in first, and here the OEM's recovery mode (like in your router) could be entered.
2) Openwrt pre-init. Early boot process. Starting essential services and checking failsafe trigger.
3) Openwrt full boot. Starting firewall, wifi, progamming switch, dhcp etc.

I would guess that for you the correct moment is when led changes from green to off and the few seconds before and after that. But that is pure guess :-(


(I have recovered my own wndr3700 about a dozen times using the Netgear OEM recovery mode, and have also visited failsafe several times.)

SantaFox wrote:

2) If I start router pressing "reset" button, it start flashing power led with yellow and green colors alternately. However, there is no ping to 192.168.1.1 as well. Actually, I have no idea what is this mode

That sounds like the Linksys "Download mode" mentioned in wiki:
http://wiki.openwrt.org/toh/linksys/wrt â€¦ nload.mode

Enter download mode on WRT350N v2 by disconnecting its power, then pressing the reset button (keep pressing), powering on the WRT350N, and releasing the reset button after 1-2 secs (no longer than 3secs, or exactly as soon as the LAN led switches on). The power led will now alternate between colours if done correctly.

That mode has nothing to do with Openwrt, so 192.168.1.1 is probably wrong. And apparently you need a special tool for uploading the new firmware in that mode. Read the wiki and possible download the upgrade tool.

You might also browse Linksys website for about that mode.
