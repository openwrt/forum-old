two observations:

1) there are a number of nvram entries that probably stem from earlier firmwares but not from OpenWrt.
Would recommend, after loading OpenWrt, to do a 'mtd erase nvram' and 'reboot' to start from a clean nvram.

2) probably the wl0_akm and wl0_auth_mode settings are conflicting. They are synonyms in recent OpenWrt releases, so it suffices to set one of them and not have the other (nvram unset wl0_auth_mode ; nvram commit). As auth_mode happens to get read after akm, it will undo akm's effect as you have set auth_mode to 'none' and akm to 'psk'

the country settings seem ok; you may check with 'wl country' whether the radio has indeed been set to ALL.
Am using > 10 channel numbers here so it should be possible.
Use of nas is in the normal OpenWrt documentation under setting up WPA-PSK. It's a copyrighted package (see non-free ipkg packages) that takes care of the wpa/wpa2 key negotiation and setting of proper keys in the encryption/decryption hardware.   An open-source alternative 'wpa_supplicant' is broken in its communication with the current broadcom radio driver wl.o.
