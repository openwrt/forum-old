anontrol
21 Apr 2008, 21:10

Hi,

I was trying to follow the instruction for loading openwrt on rb1xx equipment here:

   http://rb1xx.ozo.com/doku.php

after getting the dhcp/tftp server set up, the routerboard flash erase and the routerboard start config to set to boot off of ethernet using hdcp ptotocol, everything seems to go soothly with asterisks flowing.... but then about halfway through the load, I get "falling back to small packet size" and then "kernel loading failed" followed by a reboot.

Any ideas about what might be going on?

--Edit--

Ok... I figured it out... there was another DHCP server on the segment handing out addresses before mine.  All those asterisks were indications of failure (looks like the firmware reverts to smaller packets after a certain amount of failure).

--Edit--

One little change to those instructions at:
   
   http://rb1xx.ozo.com/doku.php

In the "RouterBoard linux install sdk" section where it says:

   host rb500 {
     hardware ethernet 00:09:22:22:22:00;
     filename "/linuxrc";
     fixed-address 192.168.0.10;
   }

the filename should be set to "openwrt-adm5120-2.6-vmlinux.elf" (maybe because I was working with an rb133c without a CF slot instead of an rb500?)

robrob
24 Apr 2008, 17:13

I had the same error message and it was caused by the RB's RouterBoot not being able to contact the tftp server when trying to load the .elf boot image, so you probably need to change your dhcp settings. The following dhcp settings work for me:

host rb133 {
     hardware ethernet 00:09:22:22:22:00;           # The RB133's #1 ethernet port MAC
     filename "openwrt-adm5120-2.6-vmlinux.elf"; # This boot image file is located in the tptp root directory
     next-server 192.168.?.?                                # The ip address of the tftp server
     fixed-address 192.168.0.10;                          # This ip address  is assigned to the RB133
   }

Make sure you select boot protocol dhcp in the RB133's RouterBoot setup.

Good luck,

Rob
