Possibly you don't need to take out anything. When you compile your own squashfs image, you can have the packages preinstalled in the squashfs partition. Squashfs is LZMA compressed so you'll probably save half the space your programs would take when installed to the JFFS2 partition (using ipkg install). See http://wiki.openwrt.org/OpenWrtDocs/Ins â€¦ 2f388a8790

Building an image is quite easy, see http://wiki.openwrt.org/ImageBuilderHowTo

/tmp is stored in RAM, so anything installed there will be gone in the next boot.

keitsi
10 Apr 2007, 23:30

There is at least a limit cause by nvram overhead (4MB - nvram), but I don't know if there is anything else in addition to that.

I have successfully installed an image of size 3182592 bytes to WRT54GL v1.1 using tftp. Also worked on Buffalo WHR-HP-G54S. The rule of thumb is that if you install an image via tftp, you can't brick your router as you can always reinstall using the same method (as long as your image doesn't turn off boot_wait or anything similar).
