rajl

The OpenWRT Wiki should answer all of your questions. 

Any router that uses a Broadcom chipset will have spotty wireless support at best.  For example, the Netgear R7000 has no wireless support now because there is no open source driver available for that Broadcom chipset.  Similarly, the Netgear R6300 only supports 802.11 b/g.  Other routers that use Broadcom chipsets have similar issues unless they are older hardware supported by the bcm43 Linux driver.

Broadcom does make a proprietary binary Linux driver available, which is how DD-WRT is able to support these routers.  However, I understand that this driver is compiled for use with much older versions of the Linux kernel (i.e. 2.6.something) which uses a different ABI than the more current kernel version used by OpenWRT.  For this technical reason, it's unlikely to see the Broadcom driver incorporated into the OpenWRT mainline soon.  OpenWRT also prefers to avoid closed-source drivers, so even if Broadcom recompiled the binary drivers to support the more current Linux kernel ABI, I doubt it would be incorporated.

For those reasons, I would steer clear of the Netgear's at this time if you want to run OpenWRT.  If you can find version 2.0 of the TP-Link Archer C7, I would go with that because it appears to be the best supported at this time.  The Linksys WRT1200AC and Linksys WRT1900AC are supported, but those builds still appear to be a work in progress from the forum threads.
