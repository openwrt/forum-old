As general rule you can load only modules compiled with your kernel.
In my experience I had many problems with foreign modules even if compiled from the same trunk but with slightly different options. I had hangs, kernel panics, symbol missings.
I suggest compile all possible kernel modules (as 'M', not '*') when you compile openwrt kernel. This guarantees you have everything compatible in case you need it

http://wiki.openwrt.org/doc/howto/usb.storage


Required Packages for USB Storage

When your USB device is properly recognised by the system, using the proper driver kernel packages listed in Basic USB support, the following packages facilitate USB storage support:

    kmod-usb-storage required Kernel support for USB Mass Storage devices
    kmod-fs-<file_system> required the filesystem you formated your partition in
    kmod-usb-storage-extras optional Kernel support for some more drivers, such as for SmartMedia card readers
    block-mount optional & recommended Scripts used to mount and check block devices (filesystems and swap) and hotplug capability (recognition when device is plugged in). This package is required for using USB storage with fstab configuration, see UCI fstab configuration.
    kmod-scsi-generic FIXME probably required Any mass storage is a generic SCSI device.
